Our Adventures at Google I/O 5/19/2011 8:31 AM by Alex Ninneman & TC Follansbee

I want to preface this by saying "I am a programmer, not a blogger". So if this ends up being dry and boring, I apologize, but hey, I tried.

We'll start by giving you a rundown on exactly what Google I/O entails. Every year for the past 4 years, Google has put on a conference in San Francisco to showcase some of the new technologies that they are working on building and distributing. Basically, if you want to know what the biggest news in tech is going to be for the next 6 mounts, pay attention!

Google invites web/mobile developers that strongly leverage Google technologies to come and preview some of the new tools that they are building, including both hardware and software. Lately the biggest buzz for Google has been their impecible impact on the smart phone market with the Android platform.

This year the biggest anouncements at Google I/O were the introduction of the new Android operating system that is optimized for tablets. Google was so excited to be releasing the new version of Android for tablets that they wanted to give all their developers a leg up on building the latest applications by giving out 5000 FREE Samsung Galaxy 10.1 tablets to the conference goers.

Along the lines of the Android platform, Google announced the BETA release of Google Music. This service will allow users to load/purchase music into the cloud and stream music directly onto their Android phones, tablets, and personal computers. So no more worrying about running out of space on your phone to store all those pesky Lady Gaga songs. Unfortunately, this service is only available to the conference goers.

Guess what, the streaming doesn't stop there! Google has appended their already existing Android Market to accomodate streaming movies. Users can rent a movie at a low rate. The user will then have 30 days to watch the movie that they have rented (similar to Netflix), however, once a user starts watching the movie they only have 24 hours to finish watching the film.

Samsung Galaxy Tab 10.1

It hasn't been any secret that Google has been developing ChromeOS. ChromeOS is a new operating system that is meant to cater to the netbook, internet user. It will run on lightweight, low-powered netbooks and consist of a beefed up version of the common Chrome browser. Chrome has been well known as the most powerful browser in todays modern computing. Leveraging the ability to do web application gaming, including the announcement of Angry Birds on the Chrome Web Store. For me, as a developer, my first question - "How do I build web applications if I can only access web sites?". Thankfully there are numerous developers that are hard at work on solutions for doing cloud web development:

  • Cloud9 - Cloud IDE that works for the main interpreted languages (Javascript, CoffeeScript, PHP, Ruby, and Python). This one's my favorite.
  • CodeRun - Browser based cloud IDE that allow code completion for C#. The support for this project seems to be a little behind. No Razor for C# =(
  • Kodingen - Another Cloud Development Environment that supports the main interpreted languages.

I think ChromeOS is going to change the way that we do computing, but probably not for another decade. The enterprise solutions are the things that we will never be able to step completely into the cloud for.

Sessions

HTML5 Showcase for Web Developers: The Wow and the How

I attended this session to gain some information on utilizing the built in database tools, File System APIs, audio, video, and canvas graphics that come as part of HTML5. In HTML5 the file input has been extended with an attribute called "multiple". Using the "multiple" attribute, the user is allowed to upload a group of files, folder of files, or a ZIP archive. The browser then has the ability to upload and manipulate multiple files at once. This will play a key part in implementing file browsers in web applications. Another great feature included in HTML5 is the ability to drag and drop files from your local system into the web application. The new specification for CSS3 allows you to do 3D grpahics along with SVG and the canvas element. The cool thing is that we can use CSS (the language that we've been using to style websites for 10 years) and build 3D graphic intensive applications using a common languages. Just don't try to make it work in IE.

Connecting People and Places ::: Location Based App development using Google APIs

This is actually two different sessions that I attended, but I'm going to sum them up as one here. They both were focused on the same technology, Google Places API. This API allows developers to build applications that can talk directly to Google Places and use the API to get Places data for populating geo-location features (Foursquare, GoWalla, and Facebook Places).

HTML5 Today with Google Chrome Frame

Chrome Frame has been around for a long time, I feel it just has never been utilized to its full potential. Most developers don't implement it because the user must install the application. Until recently, you had to have Administrative Access to your computer in order to install Chrome Frame, so on the enterprise level, it was unreachable. I installed it about a year back and remember building an application a couple days after. When I develop an application, I'll build it on Chrome and then "make it work" on IE. So when I launched IE to see how everything was, I was amazed that the site looked exactly the same as it did in Google Chrome, until I realized that Chrome Frame was the culprit. I'm happy that Google has taken the initiative to clean up on Microsoft's mistakes.

Web Fonts are changing the Web. Learn Why.

At Google I/O 2010, Google announced the release of Google Web Fonts. Typography is something that has been supremely lacking in the web. Typically users were limited to Arial, Courier New, Georgia, Times, Verdana, and Trebuchet. Which in my mind and that of most typographers, is truly disturbing. As a matter of fact, right now you are reading this article in the Ubuntu font face powered by Google Web Fonts. Typography can be very powerful if used properly. It's a very rich design aesthetic that has been under-utilized until now.

GWT + HTML5: A web developers dream!

The title doesn't lie, I was gitty walking into this session. GWT is short for Google Web Toolkit, which can be used in Java, Python, and soon Go applications. GWT gives you access to UI components and server-side functions that help you optimize for HTML5 development. There are built in libraries for checking browser compatibility, which allows you to load the page with different features configured for different browser versions.

Writing Web Apps in Go

Web back-end languages tend to be a matter of "Speed, reliability, or simplicity: pick one". Not any more. The Go Programming Language is a rock-solid, productive, and high-performance platform for building web applications (and other types of software). This session briefly went over what it takes to set up you Go environment and then get started writing some applications. The Go Team announced the support of Go on the Google App Engine.
